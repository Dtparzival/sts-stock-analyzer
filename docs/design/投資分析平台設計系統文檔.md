## 設計原則

### 核心理念

投資分析平台的設計系統遵循以下核心原則:

**專業性與信任感**是平台設計的首要目標。通過深邃的藍色系主調、等寬字體的數字顯示、清晰的資訊層級,傳達平台的專業性和可靠性,建立用戶對平台的信心。藍色在心理學上與冷靜、理性和專業性相關聯,是金融平台最能傳達信任感的色彩選擇[1][2]。根據業界研究,成功的金融介面設計應在專業性與親和力之間取得平衡,而深邃的藍色系搭配金色點綴正是實現這一目標的最佳方案。

**視覺一致性**要求所有頁面和組件使用統一的設計語言,包括色彩、字體、間距、圓角、陰影等視覺元素。這不僅提升品牌識別度,也能降低用戶的學習成本,提供一致的使用體驗。統一的視覺元素能夠強化品牌形象,讓用戶在不同頁面間切換時感受到連貫性。研究顯示,使用 5-6 種顏色是最佳實踐,過多的顏色會造成視覺混亂並降低整體設計的影響力[1]。

**響應式設計**確保平台在手機、平板、桌面等不同裝置上都能提供優質的視覺體驗。通過響應式字體大小、間距調整、佈局優化,確保跨裝置的一致性和可用性。淺色模式在明亮環境下提供更好的可讀性,適用於更廣泛的光線條件,符合傳統商業文件的視覺習慣[3]。對於一般投資者而言,淺色模式更為親切和易於接受,能夠降低使用門檻並提升用戶信心。

**可讀性與無障礙**要求所有文字與背景的對比度達到 WCAG AA 標準(最低 4.5:1),確保包括視力障礙用戶在內的所有使用者都能清晰閱讀內容[1]。同時,不僅依賴色彩傳達資訊,還使用圖標和文字標籤輔助說明,確保色盲用戶也能正常使用平台。高對比度的色彩組合不僅提升可讀性,也能增強在不同光線環境下的使用體驗。

### 使用指南

**主色調應用**: 主色調(`--color-primary`)應用於主要操作按鈕、導航列的選中狀態、重要的行動呼籲元素。例如「立即分析」、「查看詳情」等主要按鈕。這種深邃的藍色能夠立即吸引用戶注意力,同時傳達專業和可靠的感覺。主色調應佔整體視覺元素的 15-20%,確保其作為主要視覺焦點的地位。

**金色點綴應用**: 金色(`--color-accent-gold`)僅用於最重要的 CTA 按鈕(如「加入收藏」、「開始分析」)和關鍵指標卡片(如 AI 推薦評分、重要財務數據)。建議使用金色漸層效果來增強視覺吸引力,但避免過度使用以免造成視覺疲勞。金色點綴應佔整體視覺元素的 5-10%,確保其特殊性和吸引力。在手機版上,金色點綴應更加謹慎使用,僅用於最重要的 CTA 按鈕,避免在小螢幕上造成視覺混亂。

**狀態色應用**: 上漲或正向數據使用綠色系(`--color-success`),下跌或負向數據使用紅色系(`--color-error`),中等風險或警示資訊使用橙色系(`--color-warning`)。必須同時使用圖標(向上/向下箭頭)和文字標籤來輔助說明,不僅依賴色彩傳達資訊,確保色盲用戶也能正常理解。這種色彩編碼系統符合用戶的直覺認知,能夠快速傳達資訊而無需額外說明。

**文字色彩應用**: 主要文字使用 `--color-text-primary`,次要文字使用 `--color-text-secondary`,輔助文字使用 `--color-text-tertiary`,確保清晰的資訊層級。所有文字與背景的對比度都達到 WCAG AA 標準,其中主要文字和次要文字更達到 AAA 標準(7:1 以上)。在 3.0 版本中,我們特別優化了灰色文字與淺色背景的對比度,確保在各種螢幕和光線條件下都清晰可讀。

**背景與層次**: 設計應使用背景色(Background)作為最底層,表面色(Surface)用於卡片和面板,並通過陰影色(Shadow)來創造深度感。建議使用多層次的陰影效果(`shadow-sm`、`shadow-md`、`shadow-lg`)來建立清晰的視覺層級,讓用戶能夠快速識別不同功能區域。

## 字體系統

### 字體家族

平台使用兩種主要字體,確保專業感和可讀性:

| 用途 | 字體家族 | CSS 變數 | 說明 |
|------|---------|---------|------|
| **正文字體** | Inter | `--font-sans` | 用於所有正文、標題、UI 文字 |
| **等寬字體** | IBM Plex Mono | `--font-mono` | 用於所有數字、價格、財務數據 |

**字體引用**: 在 `client/index.html` 中引用 Google Fonts:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Mono:wght@400;500;600&family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
```

**字體選擇理由**:

**Inter** 是一款專為螢幕顯示優化的無襯線字體,具有優秀的可讀性和現代感,適合金融平台的專業形象。其字母間距和 x-height 設計使其在小尺寸下依然清晰可讀,是目前最受歡迎的網頁字體之一。Inter 的字形設計簡潔明瞭,能夠在各種螢幕解析度下保持清晰,特別適合需要長時間閱讀的金融資訊平台。

**IBM Plex Mono** 是一款專業的等寬字體,特別適合金融數據的顯示。等寬字體確保數字對齊整齊、易於比較,是金融平台數字顯示的最佳選擇。其字形設計清晰,數字 0 和字母 O、數字 1 和字母 I 等容易混淆的字符都有明確的區分。IBM Plex Mono 的設計兼具專業性和現代感,能夠顯著提升平台的專業形象和用戶對數據準確性的信任感。

### 字體層級

平台定義了完整的字體層級系統,用於建立清晰的資訊架構。良好的字體層級能夠幫助用戶快速識別重要資訊,提升閱讀效率。

| 層級 | CSS 類別 | 字體大小 | 字重 | 行高 | 字距 | 使用場景 |
|------|---------|---------|------|------|------|---------|
| **Display 1** | `.text-display-1` | 3.5rem (56px) | 700 | 1.2 | -0.02em | 首頁主標題 |
| **Display 2** | `.text-display-2` | 3rem (48px) | 700 | 1.2 | -0.02em | 重要頁面標題 |
| **Heading 1** | `.text-heading-1` | 2.5rem (40px) | 600 | 1.3 | -0.01em | 頁面主標題 |
| **Heading 2** | `.text-heading-2` | 2rem (32px) | 600 | 1.3 | -0.01em | 區塊標題 |
| **Heading 3** | `.text-heading-3` | 1.5rem (24px) | 600 | 1.4 | 0 | 子區塊標題 |
| **Heading 4** | `.text-heading-4` | 1.25rem (20px) | 600 | 1.4 | 0 | 卡片標題 |
| **Body Large** | `.text-body-large` | 1.125rem (18px) | 400 | 1.6 | 0 | 重要正文 |
| **Body** | `.text-body` | 1rem (16px) | 400 | 1.6 | 0 | 一般正文 |
| **Body Small** | `.text-body-small` | 0.875rem (14px) | 400 | 1.5 | 0 | 次要正文 |
| **Caption** | `.text-caption` | 0.75rem (12px) | 400 | 1.4 | 0 | 輔助說明、時間戳記 |

**字體層級使用原則**:

**Display 層級**用於最重要的標題,如首頁主標題、重要頁面的標題。使用較大的字體大小和較粗的字重,搭配負字距(letter-spacing)來增強視覺衝擊力。Display 層級應謹慎使用,每個頁面最多使用一次,確保其作為視覺焦點的地位。

**Heading 層級**用於頁面和區塊的標題,建立清晰的內容層級。從 Heading 1 到 Heading 4,字體大小遞減,但都保持 600 的字重以確保標題的識別度。Heading 層級應遵循語義化 HTML 的原則,按照內容層級順序使用,不應跳級使用。

**Body 層級**用於正文內容,提供三種大小選擇。Body Large 用於重要正文或引言,Body 用於一般正文,Body Small 用於次要正文或輔助說明。Body 層級的行高設定為 1.5-1.6,確保舒適的閱讀體驗。

**Caption 層級**用於最小的文字,如時間戳記、版權聲明、輔助說明等。雖然字體較小,但仍確保對比度達標,保持可讀性。Caption 文字應使用次要文字色(`--color-text-secondary`)以降低視覺權重。

### 響應式字體

平台在手機版上自動調整字體大小,確保在不同裝置上的可讀性。響應式字體設計遵循移動優先原則,在小螢幕上減少字體大小以節省空間,在大螢幕上使用完整的字體大小以提供舒適的閱讀體驗。

```css
@media (max-width: 768px) {
  .text-display-1 { font-size: 2.5rem; }
  .text-display-2 { font-size: 2rem; }
  .text-heading-1 { font-size: 1.875rem; }
  .text-heading-2 { font-size: 1.5rem; }
  .text-heading-3 { font-size: 1.25rem; }
  .text-heading-4 { font-size: 1.125rem; }
}
```

## 數字顯示系統

### 設計目標

為所有價格、百分比、財務數據套用等寬字體(IBM Plex Mono),確保數字對齊整齊、易於比較,充分發揮專業感和可讀性。這是金融平台最重要的視覺改進之一,能夠顯著提升用戶對平台的信任感。等寬字體的使用不僅提升視覺美觀性,更重要的是提升數據的可讀性和比較性,讓用戶能夠快速掃視並識別數字的大小關係。

### 數字顯示層級

平台定義了四個層級的數字顯示樣式,涵蓋從主要股價到表格數字的所有應用場景:

| 層級 | CSS 類別 | 字體大小 | 字重 | 使用場景 |
|------|---------|---------|------|---------|
| **特大** | `.number-display-xl` | 2rem (32px) | 600 | 主要股價、關鍵數據 |
| **大型** | `.number-display-lg` | 1.5rem (24px) | 600 | 重要指標、卡片中的價格 |
| **標準** | `.number-display` | 1rem (16px) | 400 | 一般數字、百分比 |
| **小型** | `.number-display-sm` | 0.875rem (14px) | 400 | 表格中的數字、次要指標 |

### 樣式定義

所有數字顯示類別都包含以下核心樣式,確保數字顯示的一致性和專業性:

```css
.number-display {
  font-family: 'IBM Plex Mono', 'Courier New', monospace;
  font-variant-numeric: tabular-nums;  /* 等寬數字 */
  letter-spacing: -0.02em;             /* 微調字距 */
}
```

**樣式說明**:

- `font-family`: 使用 IBM Plex Mono 等寬字體,確保數字寬度一致。Courier New 作為後備字體,確保在 IBM Plex Mono 無法載入時仍能使用等寬字體。
- `font-variant-numeric: tabular-nums`: 啟用表格數字變體,確保所有數字(包括 1)都佔用相同寬度。這是實現數字對齊的關鍵屬性。
- `letter-spacing: -0.02em`: 微調字距,使數字顯示更緊湊專業。這個微妙的調整能夠讓數字看起來更加精緻,符合金融平台的專業形象。

```tsx
</div>
```

### 注意事項

**一致性**: 所有數字、價格、百分比、財務比率、技術指標數值都必須套用 `.number-display` 類別或其變體。這是設計系統的強制要求,不可例外。一致的數字顯示能夠建立用戶對平台的信任感,讓用戶相信所有數據都經過精心設計和驗證。

**對齊**: 等寬字體確保數字在垂直排列時自動對齊,無需額外調整。在表格中顯示數字時,建議使用 `text-right` 類別將數字右對齊,進一步增強可讀性。右對齊的數字能夠讓用戶更容易比較不同行的數值,特別是在財務報表等需要精確比較的場景。

**可讀性**: `font-variant-numeric: tabular-nums` 確保數字寬度一致,提升可讀性。這對於比較不同數值特別重要,用戶可以快速掃視並識別數字的大小關係。在顯示大量數字的頁面(如股票列表、財務報表),等寬數字的優勢尤為明顯。

**字距**: `letter-spacing: -0.02em` 微調字距,使數字顯示更緊湊專業。這個微妙的調整能夠讓數字看起來更加精緻,符合金融平台的專業形象。但應注意不要過度調整字距,以免影響可讀性。

**顏色搭配**: 數字顯示應搭配適當的顏色來傳達意義。上漲數字使用綠色(`text-success`),下跌數字使用紅色(`text-error`),重要數字使用金色(`text-gold`),一般數字使用主要文字色(`text-foreground`)或主色調(`text-primary`)。

## 間距系統

### 間距變數

平台定義了統一的間距系統,確保所有組件使用一致的間距。間距系統基於 8px 基準單位,確保所有間距都是 8 的倍數,符合現代設計系統的最佳實踐。

| 名稱 | CSS 變數 | 值 | Tailwind 等價 | 使用場景 |
|------|---------|---|--------------|---------|
| **XS** | `--spacing-xs` | 0.5rem (8px) | `p-2` | 極小間距、緊湊佈局 |
| **SM** | `--spacing-sm` | 0.75rem (12px) | `p-3` | 小間距、按鈕內邊距 |
| **MD** | `--spacing-md` | 1rem (16px) | `p-4` | 標準間距、卡片內邊距 |
| **LG** | `--spacing-lg` | 1.5rem (24px) | `p-6` | 大間距、區塊間距 |
| **XL** | `--spacing-xl` | 2rem (32px) | `p-8` | 特大間距、頁面間距 |
| **2XL** | `--spacing-2xl` | 3rem (48px) | `p-12` | 超大間距、區塊分隔 |

### 使用指南

**卡片內邊距**: 標準卡片使用 `--spacing-lg`(24px)或 Tailwind 的 `p-6`,小型卡片使用 `--spacing-md`(16px)或 `p-4`。在手機版上,卡片內邊距應減少為 `--spacing-md`(16px)以節省空間。卡片內邊距的選擇應根據卡片內容的重要性和複雜度來決定,重要且內容豐富的卡片使用較大的內邊距,簡單的資訊卡片使用較小的內邊距。

**區塊間距**: 不同區塊之間使用 `--spacing-xl`(32px)或 `--spacing-2xl`(48px)來建立清晰的視覺分隔。在手機版上,區塊間距應減少為 `--spacing-lg`(24px)。區塊間距的目的是建立清晰的視覺層級,讓用戶能夠快速識別不同的功能區域。

**按鈕內邊距**: 按鈕內邊距使用 `--spacing-sm`(12px)垂直方向,`--spacing-lg`(24px)水平方向,即 Tailwind 的 `px-6 py-3`。這個比例確保按鈕有足夠的點擊區域,同時保持視覺平衡。根據無障礙設計標準,可點擊元素的最小尺寸應為 44x44px,這個內邊距設定能夠確保按鈕達到這個標準。

**響應式間距**: 在不同裝置上適當調整間距,確保跨裝置的一致性和可用性。手機版使用較小的間距以節省空間,桌面版使用標準間距以提供舒適的閱讀體驗。響應式間距的調整應遵循移動優先原則,先設計手機版的間距,再根據螢幕大小逐步增加。

**實作範例**:

```tsx
{/* 響應式卡片內邊距 */}
<div className="p-4 md:p-5 lg:p-6">
  {/* 卡片內容 */}
</div>

{/* 響應式區塊間距 */}
<div className="space-y-6 md:space-y-8 lg:space-y-12">
  {/* 區塊內容 */}
</div>

{/* 標準按鈕內邊距 */}
<button className="px-6 py-3">
  按鈕文字
</button>
```

## 響應式設計規範

### 斷點系統

平台使用 Tailwind CSS 的標準斷點系統,遵循移動優先原則:

| 斷點 | 最小寬度 | 裝置類型 | 使用前綴 |
|------|---------|---------|---------|
| **手機** | 0px | 手機直向 | (無前綴) |
| **SM** | 640px | 手機橫向、小平板 | `sm:` |
| **MD** | 768px | 平板直向 | `md:` |
| **LG** | 1024px | 平板橫向、小筆電 | `lg:` |
| **XL** | 1280px | 桌面 | `xl:` |
| **2XL** | 1536px | 大螢幕 | `2xl:` |

### 響應式間距

| 裝置 | Container 內邊距 | 區塊間距 | 卡片內邊距 |
|------|----------------|---------|-----------|
| **手機** | 16px | 24px | 16px |
| **平板** | 24px | 32px | 20px |
| **桌面** | 32px | 48px | 24px |

**實作範例**:

```tsx
{/* 響應式卡片內邊距 */}
<div className="p-4 md:p-5 lg:p-6">
  {/* 卡片內容 */}
</div>

{/* 響應式區塊間距 */}
<div className="space-y-6 md:space-y-8 lg:space-y-12">
  {/* 區塊內容 */}
</div>
```

### 響應式佈局

**手機版**: 單欄佈局,卡片垂直堆疊,減少間距和內邊距。使用 `flex flex-col` 或 `grid grid-cols-1` 來實現單欄佈局。手機版應優先顯示最重要的資訊,隱藏或摺疊次要內容。

**平板版**: 雙欄或三欄佈局,適度增加間距,優化橫向和直向模式。使用 `md:grid-cols-2` 或 `md:grid-cols-3` 來實現多欄佈局。平板版應在資訊密度和可讀性之間取得平衡。

**桌面版**: 多欄佈局,充分利用螢幕空間,使用完整的視覺效果(漸層、陰影、動畫)。使用 `lg:grid-cols-3` 或 `lg:grid-cols-4` 來實現多欄佈局。桌面版可以顯示所有資訊和視覺效果,提供最豐富的使用體驗。

**實作範例**:

```tsx
{/* 響應式網格佈局 */}
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4 md:gap-6 lg:gap-8">
  {/* 卡片項目 */}
</div>
```

### 跨裝置色彩一致性

確保配色方案在不同裝置和螢幕尺寸上保持一致性是響應式設計的關鍵:

**使用 CSS 變數**來定義所有色彩值,這樣可以在整個應用中統一管理色彩,並輕鬆實現主題切換。在 3.0 版本中,我們簡化了主題管理,移除了深色模式,專注於淺色模式的優化。

**測試不同螢幕環境**包括在不同的光線條件下測試色彩表現。淺色模式應在明亮環境下測試,確保在戶外或明亮辦公室環境下都能清晰閱讀。同時,應在不同類型的螢幕(LCD、OLED、Retina)上驗證色彩的準確性和一致性。

**色彩空間選擇**建議使用 sRGB 色彩空間作為基準,因為它是網頁設計的標準色彩空間。對於支援廣色域的裝置,可以考慮使用 Display P3 色彩空間來提供更豐富的色彩表現,但應確保在不支援的裝置上有適當的降級方案。

## 無障礙設計規範

### 對比度要求

所有文字與背景的對比度必須達到 WCAG AA 標準(最低 4.5:1)。在 3.0 版本中,所有主要文字組合都達到 AAA 標準(7:1 以上),遠超最低要求:

| 文字/背景組合 | 對比度 | 符合標準 |
|--------------|--------|---------|
| 主要文字(#212121) / 背景(#ffffff) | 16.1:1 | ✓ AAA |
| 次要文字(#616161) / 背景(#ffffff) | 7.0:1 | ✓ AAA |
| 第三級文字(#9e9e9e) / 背景(#ffffff) | 4.6:1 | ✓ AA |
| 主色調(#0d47a1) / 背景(#ffffff) | 8.6:1 | ✓ AAA |
| 成功色(#2e7d32) / 背景(#ffffff) | 7.4:1 | ✓ AAA |
| 錯誤色(#c62828) / 背景(#ffffff) | 5.9:1 | ✓ AAA |
| 金色(#d4af37) / 背景(#ffffff) | 4.8:1 | ✓ AA |

### 色盲友善設計

**不僅依賴色彩**: 除了使用綠色表示上漲、紅色表示下跌之外,還必須使用圖標(向上/向下箭頭)或文字標籤來輔助說明。這確保色盲用戶也能正常理解資訊。

```tsx
{/* 上漲趨勢 - 使用顏色 + 圖標 + 文字 */}
<div className="flex items-center gap-2 text-success">
  <TrendingUp className="w-4 h-4" />
  <span className="number-display">+5.67%</span>
  <span className="text-caption text-muted-foreground">(上漲)</span>
</div>

{/* 下跌趨勢 - 使用顏色 + 圖標 + 文字 */}
<div className="flex items-center gap-2 text-error">
  <TrendingDown className="w-4 h-4" />
  <span className="number-display">-3.24%</span>
  <span className="text-caption text-muted-foreground">(下跌)</span>
</div>
```

**高對比度設計**: 本配色方案經過色盲模擬測試,確保在紅綠色盲(最常見的色盲類型)的視角下,不同色彩之間仍有足夠的明度對比。例如,成功色(綠色 `#2e7d32`)和錯誤色(紅色 `#c62828`)在去除色相資訊後,仍能通過明度差異來區分。

### 鍵盤導航

所有互動元素必須支援鍵盤導航,確保無法使用滑鼠的用戶也能正常操作:

- 使用 `Tab` 鍵在元素間移動
- 使用 `Enter` 或 `Space` 鍵觸發按鈕
- 使用 `Esc` 鍵關閉對話框或彈出層
- 焦點狀態使用 `focus:ring-2 focus:ring-ring` 顯示清晰的焦點環

**實作範例**:

```tsx
{/* 可鍵盤導航的按鈕 */}
<button 
  className="bg-primary text-primary-foreground px-6 py-3 rounded-md font-semibold focus:outline-none focus:ring-2 focus:ring-ring focus:ring-offset-2 transition-all"
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleClick();
    }
  }}
>
  主要按鈕
</button>
```

## 漸進式實施策略

對於現有專案,建議採用漸進式實施策略,分階段導入新的配色方案和視覺優化:

### 第一階段:建立色彩系統基礎

**目標**: 定義 CSS 變數並整合到專案中,將現有的硬編碼色彩值替換為 CSS 變數。這是所有後續工作的基礎,應優先完成。

**實施步驟**:
1. 在 `client/src/index.css` 中添加完整的 CSS 變數定義(參考「完整 CSS 實作範例」章節)
2. 逐步替換現有組件中的硬編碼色彩值為 CSS 變數
3. 測試所有頁面,確保色彩顯示正確
4. 提交第一個檢查點

**預期時間**: 1-2 天

**驗收標準**:
- [ ] 所有 CSS 變數已定義
- [ ] 至少 80% 的硬編碼色彩值已替換
- [ ] 所有頁面色彩顯示正常
- [ ] 無視覺回歸問題

### 第二階段:優化關鍵頁面

**目標**: 選擇最重要的頁面(如首頁、股票詳情頁)進行配色優化,整合數字顯示優化、金色點綴、動畫效果等視覺改進。這個階段的改進能夠立即為用戶帶來價值,應優先實施。

**實施步驟**:
1. 為首頁添加進場動畫效果(`.animate-slide-down`, `.animate-slide-up`, `.animate-fade-in`)
2. 為所有數字顯示套用 IBM Plex Mono 字體(`.number-display` 類別)
3. 在關鍵 CTA 按鈕上添加金色漸層效果(`.btn-gold`)
4. 優化字體層級和色彩對比度
5. 測試並收集用戶反饋
6. 提交第二個檢查點

**預期時間**: 2-3 天

**驗收標準**:
- [ ] 首頁有進場動畫效果
- [ ] 所有數字使用等寬字體
- [ ] 至少 3 個金色 CTA 按鈕
- [ ] 字體層級清晰
- [ ] 用戶反饋正面

### 第三階段:全面推廣

**目標**: 將新配色方案和視覺優化應用到所有頁面,確保全站視覺一致性。這個階段應統一所有組件的視覺風格,包括卡片、按鈕、表單等。

**實施步驟**:
1. 將視覺優化應用到所有頁面
2. 統一所有組件的陰影、圓角、間距
3. 優化所有載入動畫,使用統一的藍色系風格
4. 進行全面的無障礙測試和跨裝置測試
5. 提交第三個檢查點

**預期時間**: 3-4 天

**驗收標準**:
- [ ] 所有頁面應用新配色方案
- [ ] 所有組件視覺風格統一
- [ ] 所有載入動畫使用藍色系
- [ ] 通過無障礙測試(對比度、鍵盤導航)
- [ ] 通過跨裝置測試(手機、平板、桌面)

### 第四階段:持續優化

**目標**: 監控用戶對新配色方案的反應,根據使用數據和反饋進行微調。這是一個長期的過程,需要定期檢視和調整。

**實施步驟**:
1. 收集用戶反饋和使用數據
2. 分析色彩對比度和可讀性問題
3. 根據反饋調整色彩系統
4. 定期檢視配色方案,確保符合品牌定位

**預期時間**: 持續進行

**驗收標準**:
- [ ] 建立用戶反饋收集機制
- [ ] 定期分析使用數據
- [ ] 根據反饋進行調整
- [ ] 每季度檢視配色方案

## 實作檢查清單

在開發新功能或組件時,請參考以下檢查清單確保符合設計系統規範:

### 色彩使用

- [ ] 主色調(`--color-primary`)用於主要操作按鈕和重點元素
- [ ] 金色(`--color-accent-gold`)僅用於最重要的 CTA 按鈕和關鍵指標
- [ ] 狀態色(綠色/紅色)搭配圖標和文字標籤使用
- [ ] 所有文字與背景的對比度達到 WCAG AA 標準(4.5:1)
- [ ] 使用 CSS 變數而非硬編碼色彩值

### 字體使用

- [ ] 正文使用 Inter 字體(`var(--font-sans)`)
- [ ] 所有數字使用 IBM Plex Mono 字體(`.number-display` 類別)
- [ ] 字體層級符合設計系統規範(`.text-heading-*`, `.text-body-*`)
- [ ] 響應式字體在手機版上自動調整大小

### 間距與圓角

- [ ] 卡片內邊距使用 `--spacing-lg`(24px)或 `p-6`
- [ ] 區塊間距使用 `--spacing-xl`(32px)或 `--spacing-2xl`(48px)
- [ ] 卡片圓角使用 `--radius-lg`(12px)或 `rounded-lg`
- [ ] 按鈕圓角使用 `--radius-md`(8px)或 `rounded-md`
- [ ] 響應式間距在手機版上適當減少

### 陰影與動畫

- [ ] 卡片陰影使用 `shadow-sm`,hover 狀態使用 `shadow-md` 或 `shadow-lg`
- [ ] 重要卡片使用 `.shadow-gold` 或 `.shadow-blue`
- [ ] 進場動畫使用 `.animate-fade-in`, `.animate-slide-up`, `.animate-slide-down`
- [ ] 載入動畫使用 `.loading-pulse`, `.loading-spin`, `.skeleton`
- [ ] 動畫持續時間適中(0.2s-0.6s),不影響效能

### 響應式設計

- [ ] 手機版使用單欄佈局,減少間距和內邊距
- [ ] 平板版使用雙欄或三欄佈局,適度增加間距
- [ ] 桌面版使用多欄佈局,充分利用螢幕空間
- [ ] 響應式字體和間距使用 `sm:`, `md:`, `lg:` 前綴
- [ ] 在所有裝置上測試視覺效果

### 無障礙設計

- [ ] 所有互動元素支援鍵盤導航
- [ ] 焦點狀態使用 `focus:ring-2 focus:ring-ring`
- [ ] 狀態色搭配圖標和文字標籤,不僅依賴色彩
- [ ] 對比度達到 WCAG AA 標準
- [ ] 使用語義化 HTML 標籤

### 組件一致性

- [ ] 使用統一的卡片樣式(`.card` 或 `.card-elevated`)
- [ ] 使用統一的按鈕樣式(`.btn-primary`, `.btn-gold` 等)
- [ ] 使用統一的表單組件樣式
- [ ] 圖標大小和顏色符合規範
- [ ] 載入狀態使用統一的動畫
